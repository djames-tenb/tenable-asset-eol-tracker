Notes
  • API credentials are encrypted at rest using a per-install key file — they are never stored in plaintext.
  • Assets are pulled via the Tenable VM Assets Export API (no 5 000-asset cap) — scoped to VM scanner sources only (NESSUS_SCAN + NESSUS_AGENT) and assets last seen within 90 days.
  • Both OS strings and installed software CPE data are matched against endoflife.date for EOL dates. 50+ products are mapped. EOL data is cached locally and survives restarts.
  • Asset tags from Tenable are fetched during sync — tag filters are available on every page (Dashboard, Asset Inventory, OS EOL, App EOL, Software Inventory) to scope any view by environment, owner, or any other tag category.
  • The Operating Systems EOL Overview dashboard is interactive — each summary card is clickable and navigates directly to the OS EOL page with the matching status and tag filter pre-applied.
  • Click any row in the Asset Inventory to open the Drill Down panel showing identity attributes, scan metadata, ACR/AES risk scores, full EOL analysis, tags, and raw software CPE inventory.
